【前置约定：向量/Embedding 不纳入角色库】
- 角色库仅负责“角色资料的结构化管理与审阅”，不内置或耦合任何向量/检索功能。
- 如需生成向量，由现有“向量库子系统”在外部完成；角色库不提供清空/重建/调参等入口。
- 若项目里已有相关向量逻辑，保持现状、保持解耦；若没有，则不新增。
- 本文其余设计与接口均与向量无关，可独立实现与演进。
角色库（重构版）— 需求说明（Draft v1）

一、背景与目标
- 背景：现有 character_state.txt 由 LLM 根据定稿章节自动生成；“角色库”单独存在、交互不统一，易空白、难维护。
- 目标：将“角色资料”统一托管在【角色库】，支持两类来源并行：
  1) 用户手动维护（优先级最高）
  2) LLM 自动抽取（等价于现有 character_state.txt 的能力）
- 用途：在章节草稿生成时，稳定提供结构化、最新且可控的角色信息，避免设定漂移、细节缺失。

二、术语与分层
- 角色（Character）：一个具名实体（人/组织/实体）。
- 资料条目（Entry）：关于角色的结构化字段集合（基本信息、关系、设定等）。
- 来源层（Source Layer）：
  - Manual（手动层）：用户编辑产生；优先级最高。
  - Auto（自动层）：定稿后由 LLM 抽取产生；用于增量补全与提示。
- 合成视图（Effective View）：根据“优先级与合并规则”产出的最终可用角色资料（供检索/提示）。

三、目录与存储设计
- 目录：<保存路径>/characters/
  - manual/               // 手动层（每个角色一个文件）
  - auto/                 // 自动层（按章节增量快照）
  - index.json            // 角色索引（ID、别名、最近更新时间、最近来源、标签）
  - schema.json           // 字段约定（可选）
- 文件格式：建议 YAML 或 JSON（UTF-8），文本域允许 Markdown。
  - 手动层：characters/manual/<character_id>.json
  - 自动层：characters/auto/<chapter_num>/<character_id>.json  （每章一份增量）
- 字段建议（可扩展）：
  - id（string，稳定 ID，默认用名字 slug 化）
  - name（string，显示名）
  - aliases（string[]，别名）
  - summary（string，人物摘要/人设画像）
  - attributes（map<string, string[]|string>，关键属性：外貌/性格/能力/目标/秘密/禁忌等）
  - relationships（map<string, string>，与其他角色的关系）
  - timeline（Event[]：{when, where, what, ref_chapter}）
  - tags（string[]）
  - source（manual|auto）
  - updated_at（ISO8601）

四、来源与采集
- 手动层（Manual）：
  - UI 可新建/编辑/重命名/归类（标签）；支持 Markdown 富文本域；保存即生效。
  - 支持从文件导入 txt/docx/markdown 并转为结构化字段（半自动：先解析→用户确认→保存）。
- 自动层（Auto）：
  - 触发：章节“定稿”后，调用抽取器（LLM）从当章文本抽取角色条目（新增/变更/关系/时间线）。
  - 结果：以 per-chapter 增量文件落在 auto/<chapter_num>/ 下；不会覆盖手动层。
  - 抽取器：提示词模板 + 解析器（JSON schema 校验），失败则降级为文本解析。

五、合并与优先级
- 生效规则（构建 Effective View 时）：
  - 以手动层为基底：字段级别覆盖优先（Manual > Auto）。
  - 自动层按章节时间顺序叠加：只补充缺失字段、或为数组字段执行去重合并。
  - 冲突策略：
    - 标量字段：若 Manual 已填，则忽略 Auto；若 Manual 空且 Auto 多候选，按最新章节优先。
    - 数组字段：并集去重，保持时间顺序；可配置“上限 N + 最近优先”。
    - 关系字段：按目标角色 ID 归并；描述冲突保留最近版本并在 UI 标注“自动覆盖来源”。
- 可回溯：允许查看某字段的来源链（manual/auto@chapter）。

六、生命周期与版本
- 角色生命周期：新建 → 富集（手动/自动）→ 使用（检索/提示）→ 维护（编辑/合并/清理）。
- 版本：
  - 手动层：每次保存生成一个版本（可选保留最近 N 个）。
  - 自动层：天然按章节分版本；支持按章回溯。

七、与“草稿生成”的集成
- 选择机制：
  - 在“参与角色”字段支持从角色库多选（搜索/标签过滤/最近出现），回填到提示词。
  - 生成提示词时，系统读取 Effective View，为每个入选角色注入：summary/关键属性/与当前章相关的近期时间线片段。
- 自动候选：
  - 在无手动选择时，可按“最近 N 章出现频次+相关性（向量检索）”自动候选 K 个角色，供用户一键采纳。
- 上下文体积控制：
  - 为每个角色做“压缩摘要”（可缓存），优先包含“与当前章剧情冲突相关的属性/关系/时间线”。

八、与向量库/检索的关系
- 入库：
  - 合成视图（Effective View）生成“角色卡片向量”，写入 vectorstore（metadata: {source:'character', character_id, active:true, version}）。
  - 手动保存或自动抽取更新时，触发向量增量更新。
- 检索：
  - 写作检索时允许一个独立的 Top-K for Characters（默认 2–3），参与到知识拼接；可与章节片段检索分开配额。

九、UI 设计（最小可用）
- 角色库主窗：
  - 左：角色列表（搜索/标签/来源过滤）+ 新建/导入/删除/移动/重命名。
  - 右：角色编辑器（多分栏：基本/属性/关系/时间线/摘要），显示“字段来源”标记。
  - 顶：合并视图开关（查看 Manual/Auto/Effective 切换）。
- 集成入口：
  - 主界面“参与角色”旁：
    - [从角色库选择] 按钮 → 弹窗多选 → 回填。
    - [最近候选] 下拉 → 基于最近频次/相关性自动推荐。

十、生成与定稿联动
- 定稿后：
  - 调用抽取器 → 写入 auto/<chapter>/… → 重建角色 Effective View → 更新向量 → 在日志/角色库右上角提示“X 处角色属性已更新（可查看差异）”。
- 失败与回退：
  - 抽取失败写入 error 日志，不影响写作；允许手动触发“重跑抽取 @某章”。

十一、权限与审计（可选）
- 审批：可将 Auto 的变更列为“待采纳”，由用户勾选后落入 Manual（一键采纳/部分采纳）。
- 审计：index.json 记录最后一次修改来源/时间/操作者（如有多用户）。

十二、性能与健壮性
- 文件粒度小（每角色/每章各一），避免大文件冲突；读合成视图时做内存缓存（LRU by project）。
- 提供批量校验工具：字段必填/ID 唯一/关系引用有效/编码 UTF-8 无乱码。

十三、迁移与兼容
- 读取旧的 character_state.txt：
  - 一次性导入到 auto/legacy/ 下（chapter=0 标记），合成视图自动可见。
- 原“角色库”目录（若存在）
  - 尝试解析旧格式，导入到 manual/ 并给出迁移报告。

十四、测试案例（要点）
- 合并规则：Manual 覆盖 Auto；数组合并去重；关系冲突取最近 auto，且保留来源标记。
- 定稿触发：写入 auto/<chapter>/… 后，Effective View 更新正确；向量增量写入成功。
- 选择入口：从角色库多选回填“参与角色”；无选择时自动候选可用。
- 迁移：导入 character_state.txt 与旧角色库成功；index.json 正确。

十五、里程碑
- M1（骨架）：目录结构/读写 API/Effective 合并/迁移脚本/最小 UI（列表+编辑器+来源标记）。
- M2（联动）：定稿抽取器接入/向量增量/“从角色库选择”入口/自动候选。
- M3（完善）：差异审阅与采纳/时间线可视化/批量校验工具/性能优化。

十六、开放问题
- 字段 Schema 固定 vs 可配置？（先内置，后放开）
- 多项目共享角色资产的需求？（后续支持导入/导出包）
- 角色与“叙事图谱”的关系：是否建立统一实体图（可在 M3 评审）

—— 补充：管理功能与界面设计（v1） ——

A. 信息架构与角色生命周期
- 实体层级：角色（Character）→ 字段组（Basic/Attributes/Relationships/Timeline/Notes/Tags）→ 字段项。
- 生命周期：新建 → 编辑（Manual）/抽取（Auto）→ 审阅合并 → 生效（Effective）→ 检索与提示 → 维护/回滚。
- 权限与来源：字段级来源标记（manual/auto@chapterN），Manual 可“锁定”（Lock）禁止被 Auto 覆盖；所有采纳动作写入审计（index.json）。

B. 管理功能（按优先级）
1) 角色 CRUD
  - 新建/重命名/删除/移动分类（支持多标签 Tag，分类为单一层次，Tag 可多选）。
  - 合并重复：按 name/aliases 相似度给出候选，用户选择 A←B 合并（保留 A 的 ID；字段按来源策略合并）。
2) 字段编辑与锁定
  - 字段类型：string、string[]、kv-map、timeline（when/where/what/ref_chapter）。
  - 锁定字段：勾选后不再被 Auto 层改写（除非用户显式“解锁”）。
3) 自动抽取审阅（Review Queue）
  - 按章节分组展示 Auto 增量：新增/变更/关系变更/时间线事件。
  - 字段级“采纳到 Manual”复选框；支持“全选采纳/仅变更采纳”。
  - 差异视图：左 Manual、右 Auto@chapterN，高亮差异；支持一键采纳或忽略。
4) 批量操作
  - 批量加/删 Tag，批量移动分类，批量锁定字段（如 summary/relationships）。
  - 批量导入（txt/docx/md）：解析 → 批量审核 → 批量落库。
5) 检索与导航
  - 顶部搜索框（Name/Alias/Tag/字段内容），支持正则/拼音首字母；快速跳转最近编辑/最近出现章节。
6) 版本与回滚
  - Manual 层保留最近 N 个版本（默认 5），支持单字段回滚与整卡回滚。
7) 与写作联动
  - “从角色库选择”弹窗：左侧列表 + 右侧预览 + 多选，支持按“最近出现/Tag/相关性”排序；确定后回填到“参与角色”。
  - 自动候选：在草稿页以灰色提示展示系统推荐（可一键采纳）。
8) 与向量库联动
  - 保存/采纳后触发向量增量更新（source:'character', character_id, active:true, version）。
  - 页面内提供“刷新角色向量”与“仅角色检索 Top-K”调试入口。

C. 存储与文件结构（细化）
- characters/
  - manual/<id>.json         // 字段：见“字段建议”；含 locked: {field: true}
  - auto/<chapter>/<id>.json // 仅包含从该章抽取到的增量字段 + meta: {chapter, extracted_at}
  - index.json               // { characters:[{id,name,aliases,tags,updated_at,latest_source,manual_locked_fields[]}], tags:[], stats:{} }
- 示例（manual/<id>.json）：
  {
    "id":"lin_yue",
    "name":"林越","aliases":["阿越"],
    "summary":"……",
    "attributes":{ "性格":["克制","记仇"],"能力":["刀术"] },
    "relationships":{ "苏锦":"同门旧识" },
    "timeline":[{"when":"第5章","what":"与苏锦重逢","ref_chapter":5}],
    "tags":["正派"],
    "locked":{"summary":true},
    "source":"manual","updated_at":"2026-03-28T10:20:00+08:00"
  }

D. 界面设计（CustomTkinter 版）
1) 主窗口布局
- 顶栏：搜索（⌘/Ctrl+K）、过滤（来源: All/Manual/Auto/Eff；Tag 多选）、按钮：新建角色/导入/合并重复/校验/刷新向量。
- 左栏（角色列表，固定宽度 320）：
  - 列表项：name + tags + 最近章节徽标；右键：重命名/删除/移动/合并。
  - 统计：总数、筛选后数量、最近更新时间。
- 右侧（编辑器，分 Tab）：
  - Basic：name/aliases/tags/summary（Markdown 编辑器）
  - Attributes：分组表格（键→多值），支持新增组与条目，条目可拖拽排序；锁定按钮。
  - Relationships：表格（target→描述），可从库内选择 target；显示联动回指。
  - Timeline：事件列表（when/where/what/ref_chapter），支持按章过滤；从章节定位跳转。
  - Review（仅当有待审条目）：Auto@chapterN → 差异视图 → 选择采纳 → 一键应用。
  - JSON（开发者）：直接查看合成后的 Effective JSON（只读），便于排查。
- 底部状态栏：当前角色 ID、来源覆盖比（Manual vs Auto）、未处理审阅条目数、向量状态。

2) 交互细节与快捷键
- 新建（Ctrl+N）、保存（Ctrl+S）、搜索（Ctrl+K）、采纳全部（Ctrl+Shift+Y）、锁定/解锁字段（L）。
- 双击左侧项 → 打开 Basic；中键新标签页（可选）。
- 悬浮显示字段来源：manual / auto@chapterN；点击来源可定位到该章节或审阅视图。

3) “从角色库选择”弹窗
- 左：可筛选列表（支持多选、排序：最近出现/相关性/名称）。
- 右：合成视图预览；勾选“注入摘要/关系/时间线片段”。
- 底：确定/取消；确定后把选中的 name 回填“参与角色”。

4) 审阅面板（Review Queue）
- 章节筛选（多选），分组展示字段差异卡片：字段名 / Manual 值 / Auto@chapter 值。
- 操作：采纳到 Manual、忽略、稍后提醒；支持“仅新增”“仅变更”。

E. 模块与 API（实现建议）
- character_store.py：
  - load_index/save_index、load_manual/save_manual、list_auto(chapter?)、save_auto、
    build_effective(character_id)、merge_effective(all?)（带缓存）。
- character_extract.py：
  - extract_from_chapter(chapter_text, chapter_num) → {id→partial_fields}
  - validate_against_schema(data)；失败降级解析器。
- character_merge.py：
  - merge(manual, auto_list, locks) → effective, diff（按前述规则）。
- ui/role_library2.py（新）：
  - UI 组件：ListPanel、EditorTabs（Basic/Attributes/Relationships/Timeline/Review/JSON）、
    SelectDialog、DiffViewer。
- integration hooks：
  - finalize_chapter_ui → trigger extract → save auto → rebuild effective → update vectorstore。
  - build_chapter_prompt → open SelectDialog（回填参与角色）；并读取 Effective 注入关键信息。

F. 校验与工具
- 校验命令：tools/validate_characters.py（检查编码/必填/引用有效/重复 ID/锁定覆盖冲突）。
- 迁移命令：tools/migrate_character_state.py（character_state.txt → auto/legacy/）。

G. 性能与缓存
- LRU 缓存：effective-by-id、最近章节的 auto-list；编辑/采纳后局部失效。
- 大规模项目：列表虚拟化（按需渲染）、搜索异步、写盘节流（debounce 1s）。

H. 风险与权衡
- 字段 schema 先固定，避免解析器复杂度爆炸；后续开放自定义。
- 审阅粒度控制：默认字段级，必要时允许条目级（timeline event）。

I. 验收口径（M1）
- 可新建/编辑/保存 Manual；能导入 character_state.txt 并生成 auto/legacy/。
- build_effective 正确体现“Manual 优先 + Auto 叠加”；UI 能切换 Manual/Auto/Effective 视图并展示来源标记。
- 从角色库选择回填“参与角色”；保存/采纳后角色向量更新；日志可追踪。
