# 角色库（用户指南）

> 目标：帮助你从“已定稿的章节”中自动抽取角色信息，并在写作中随时选用、维护与审阅。角色库独立于“向量/检索”系统，不引入向量依赖；信息量小、管理要简单。

## 开箱即用（三步）

1) 选择保存路径
- 右侧“保存路径”选择你的项目目录（例如 `F:/AI_NovelGenerator_workspace`）。

2) 定稿章节（自动抽取）
- 在“章节号”选择要定稿的章，确保编辑区文本就是最终稿。
- 点击“定稿”按钮。完成后系统会自动：
  - 抽取角色（写入 auto 层）；
  - 更新全局摘要与角色状态（legacy `character_state.txt` 仍保留）。
- 抽取日志会显示在 UI 底部日志区与 `app.log`：
  - `[角色库] Finalize Hook: 开始从第X章抽取角色…`
  - `[角色库] 抽取完成: chapter=X count=N`
  - `[角色库] 已写入自动层: chapter=X count=N`

3) 审阅并选用
- 主页上排点击“角色库”→ 打开“角色库工作台”：
  - 选“角色审阅（采纳自动抽取）”：选择章节→ 勾选字段→ 采纳到“手动层”。
  - 或选“从角色库选择（回填参与角色）”：勾选角色→ 确定后自动回填到参数页的“参与角色”。
- 从此，生成章节/草稿时将参考“参与角色”。

提示：参数页里，
- “从角色库选择”按钮也出现在“参与角色”输入框右侧，便于就地回填。

## 核心概念

- 自动层（auto）：定稿后自动抽取的增量信息，按章存放，低优先级，不覆盖手动信息。
- 手动层（manual）：你确认/维护的权威信息，最高优先级。审阅“采纳”即写入此层。
- 索引（index.json）：角色清单与最近来源，便于快速检索和展示。
- 合成视图（Effective）：运行时将 manual 优先，与 auto 历史合并后的视图，用于预览与选择。

## 界面总览与入口

- 角色库工作台（主页上排“角色库”）：
  - 角色审阅（采纳自动抽取）：按章节查看 auto 条目，按字段勾选采纳到 manual。
  - 从角色库选择（回填参与角色）：检索/多选→ 回填参数页“参与角色”。
  - 打开角色库文件夹：直达磁盘目录，方便你手动查看/编辑 JSON。
- 参数页 → 参与角色：左文本框 + 右侧“导入/从角色库选择”两个按钮。

## 触发与抽取规则

- 抽取时机：点击“定稿”后自动触发，无需手工命令。
- 识别规则（M1 简化版）：
  - 行首“`角色：<名字>`”或“`【角色】<名字>`”将被识别为角色出现。
  - 例如：`角色：林北`、`【角色】苏芷`。
- 建议写法：在本章首次引入或重要出场处加一行上述标记，便于稳定识别。
- 失败定位：
  - UI 日志若显示 `count=0`，说明本章未匹配到标记；可补充标记后再次“定稿”。

## 审阅与采纳（手动层优先）

- 选择章节后，面板列出该章抽取到的角色。
- 勾选要采纳的字段（支持：`name, aliases, tags, summary, attributes, relationships, timeline`）。
- 点击“采纳所选”，这些字段将写入 manual 层；后续抽取不会覆盖已采纳/锁定的手动字段。
- 日志：`[角色库] 采纳完成：章节=<n> 角色数=<k>`。

进阶：你可以在 `manual/<角色ID>.json` 添加 `locked` 字段（布尔或按字段粒度）来锁定关键内容，例如：
```json
{
  "id": "林北",
  "name": "林北",
  "summary": "主角，性格沉稳。",
  "locked": { "summary": true }
}
```

## 生成时如何使用角色信息

- 参数页 → “参与角色”：
  - 点击“从角色库选择”，勾选角色→ 确定后自动回填。
  - 这些角色会进入提示词构造，指导草稿/章节生成更聚焦于指定人物与关系。

## 数据存储结构

位于 `<保存路径>/characters/`：
- `auto/<章节号>/<角色ID>.json`：按章增量。
- `manual/<角色ID>.json`：手动权威信息。
- `index.json`：索引（角色列表、最近来源、更新时间）。

示例（auto 条目最小字段）：
```json
{
  "id": "林北",
  "name": "林北",
  "summary": "来自第12章的抽取",
  "source": "auto",
  "updated_at": "2026-03-28T12:34:56"
}
```

## 日志与排错

- UI 底部日志区：实时打印抽取/采纳/打开弹窗等操作信息。
- 文件日志：`app.log`（UTF‑8），可用于排查历史问题。
- 常见问题：
  1) 审阅面板为空？
     - 检查是否已对该章执行“定稿”；确认文本里存在“角色：/【角色】”标记。
  2) 已定稿但没有抽取？
     - 重新打开该章，点击“定稿”即可补抽取（不影响已存在的手动信息）。
  3) 想手动新建/编辑角色？
     - 直接编辑 `characters/manual/<角色ID>.json`，保存后即可在选择面板检索。

## 推荐工作流

写作一章 → 定稿（自动抽取） → 角色审阅（按需采纳） → 在参数页“从角色库选择”回填参与角色 → 继续生成下一章/草稿。

## 设计边界

- 模块边界：角色库不耦合向量/检索；即使你清空/重建向量库，角色库数据不受影响。
- 兼容性：不考虑旧结构兼容；以当前最合理的方案优先。
- 可扩展性：抽取器（M1）为正则版，后续可替换为 LLM 抽取而无需改动 UI 流程与存储约定。

## 快速参考（操作清单）
- 开启工作台：主页“角色库”→ 角色库工作台。
- 审阅采纳：工作台→ 角色审阅→ 选章节→ 勾字段→ 采纳所选。
- 选择回填：工作台→ 从角色库选择→ 勾选→ 确定（或在参数页右侧按钮）。
- 查看文件：工作台→ 打开角色库文件夹（或直接浏览 `<保存路径>/characters/`）。
